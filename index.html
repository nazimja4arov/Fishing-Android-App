Fishook

Overview

Fishook is a mobile application that utilizes the Maps SDK to track users’ fishing locations and assign points accordingly. We also incorporated a database to hold the fish information for both the user profile and leaderboard. Fishes will populate your profile once added to the app. Points are assigned to users based on the fish’s length and weight.

Requirements

This application was built using Kotlin in Android Studio, and it requires these dependencies to run the Maps SDK and the Room database, respectively: 

dependencies {
    // Google Maps
    implementation("com.google.android.gms:play-services-maps:18.2.0")
    implementation("com.google.maps.android:maps-compose:2.11.4")

    // Room
    implementation("androidx.room:room-runtime:2.6.1")
    implementation("androidx.room:room-ktx:2.6.1")
    kapt("androidx.room:room-compiler:2.6.1")
}

We use an Android Room database to store the user’s information.

Getting Started

This guide explains how to clone, run, and modify the app in Android Studio.

1. Requirements

  • Android Studio installed
  • Git installed
  • Emulator or physical device (API 24+)

2. Clone the Project

git clone https://github.com/nazimja4arov/Fishing-Android-App.git
cd Fishing-Android-App

3. Open in Android Studio

  • Open Android Studio > “Open an existing project”.
  • Wait for Gradle sync to finish.
  • Select an emulator/device and press Run.

4. Editing the Code

  • Edit screen UI inside screens/ folder.
  • Adjust logic and fish data inside ViewModel classes.
  • Re-run the app to see your changes.

Project Structure

  • screens/ — UI screens built with Jetpack Compose
  • navigation/Route.kt — sealed class for navigation
  • viewmodel/ — app logic + state (leaderboard, fish list)

Add Fish Screen

Screenshot: View Screenshot

This screen is used once the user is logged in to add a fish to their collection. Users must input the fish’s name (or general description of the fish if they aren’t familiar with it), the fish’s length (in inches), and the fish’s weight (in pounds). Users can add an image from their gallery to use for their fish; however, if they don’t wish to or don’t have a picture of their catch, a default image will be used.

Once the fields have been filled correctly, the fish and all of its data will be added to the list of the user’s catches. Fish are defined as a data class following this format: 

data class FishDataClass(
    val userName: String,
    val fishName: String,
    val fishImage: String?,
    val fishLength: Int,
    val fishWeight: Int
)

And fish are added to a user’s profile with this code:

fun addFish(name: String, image: String?, length: Int, weight: Int) {
    val img = image ?: defaultImage
    val newEntry = FishDataClass(
        userName = currentUserId,
        fishName = name,
        fishImage = img,
        fishLength = length,
        fishWeight = weight
    )

    val updatedList = getCurrentUserFish() + newEntry
    fish = fish + (currentUserId to updatedList)

    val user = getCurrentUser()
    if (user != null) {
        val newPoints = user.userPoints + length + weight * 2
        users = users + (currentUserId to user.copy(userPoints = newPoints))
    }
    viewModelScope.launch {
        repo.saveFish(
            FishEntity(
                userId = currentUserId,
                name = name,
                length = length,
                weight = weight,
                image = img
            )
        )
    }
}

Leaderboard Screen

Screenshot: View Screenshot

This screen displays the top scorers who use the app, listed in descending order for simplicity. To get onto the leaderboard, you have to catch a fish, click the add fish button, put in the details, and watch your score go up!

This screen takes in the user view model to pull in the player stats information for the leaderboard. It also calls the Fish repository, which stores the information for what fish are added to your profile. This also updates the total points for the user based on the fish in the database.

If you want to change how points are calculated, you would change the calculation for the newPoints variable in this code snippet: 

val user = getCurrentUser()
if (user != null) {
    val newPoints = user.userPoints + length + weight * 2
    users = users + (currentUserId to user.copy(userPoints = newPoints))
}

Conclusions

Overall, Fishook is a great app that can not only be used to get outdoors but also to connect people locally (and hopefully in the future globally)! For future updates to Fishook, we would like to sync the map location with the Add Fish screen, so that the location information gets saved to the fish repository and user profile. Another future update would include implementing Firebase for users to log in. We’ve learned so much over the past semester and want to say thank you to everyone for a good semester.
